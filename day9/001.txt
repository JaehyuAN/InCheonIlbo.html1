    Mark up? 
    1. 네트워크를 사용하는 이유 - 프린터와 저장장치 공유 -> LAN 등장
    2. LAN과 LAN이 연결 -> 인터네트워크
        네트워크간 정보 교환이 필요
        네트워크끼리 운영체제가 다름
        정보를 주고 받을 수 있는 표준의 필요성 대두 -> Mark up 등장
    3. Makr Up의 등장
    <제품>
        <회사></회사>
        <이름></이름>
    <제품>
    <></> = 태그
    4. MarkUp 언어
        1.XML(eXtensible Markup Language)
            사용자 정의 태그
            자료의 구조를 설명
        *XML 문법 (Well-formed)
            전체를 감싸는 루트 태그가 있어야한다
            태그를 열었으면 반드시 닫는다
            태그를 연 순서대로 닫아야한다
            ex. <이름>안재휴</이름>
                <나이>26</나이>
        2.HTML(HyperText Markup Language)
            태그가 표준화 (W3C, WWW)
            웹 페이지를 작성
        * react.js, vue.js를 사용할때는 xml형식의 html을 사용한다.
